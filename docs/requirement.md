 채팅 애플리케이션 개발 계획

# 1. 프로젝트 목표

본 프로젝트의 목표는 명시된 모든 기능을 안정적으로 구현하고, 효율적인 데이터 흐름 및 상태 관리를 통해 예측 가능하고 유지보수가 용이한 웹 애플리케이션을 구축하는 것입니다.

*   **완전한 기능 구현:** 로그인/회원가입부터 메시지 삭제에 이르기까지, 미리보기에 포함된 모든 기능을 완성도 높게 개발합니다.
*   **효율적인 상태 관리:** 불필요한 상태 관리를 최소화하고, 애플리케이션의 성능과 안정성을 높이기 위해 반드시 필요한 데이터만을 상태로 관리합니다.
*   **예측 가능한 데이터 흐름:** Flux 아키텍처를 도입하여 데이터의 흐름을 단방향으로 제어함으로써, 복잡한 애플리케이션에서도 데이터 흐름을 쉽게 예측하고 디버깅을 용이하게 만듭니다.
*   **중앙화된 비즈니스 로직:** React의 Context API를 활용하여 여러 컴포넌트에 흩어져 있는 비즈니스 로직을 중앙에서 관리함으로써, 코드의 재사용성을 높이고 컴포넌트 간의 결합도를 낮춥니다.

# 2. 기술 스택 및 외부 서비스

Flux 아키텍처와 Context API 사용으로 미루어 볼 때, 본 프로젝트는 React를 중심으로 한 기술 스택을 활용할 것으로 예상됩니다.

*   **아키텍처:**
    *   **Flux:** 단방향 데이터 흐름을 통해 애플리케이션의 상태 변화를 예측 가능하게 관리하는 아키텍처입니다. 이는 액션(Action), 디스패처(Dispatcher), 스토어(Store), 뷰(View)로 구성됩니다.
*   **핵심 라이브러리/API:**
    *   **React:** 사용자 인터페이스 구축을 위한 핵심 라이브러리입니다.
    *   **Context API:** React에 내장된 기능으로, props를 통해 데이터를 전달할 필요 없이 컴포넌트 트리 전체에 걸쳐 데이터를 공유할 수 있게 해줍니다. 이를 통해 비즈니스 로직을 중앙에서 관리합니다.
*   **외부 서비스:**
    *   **Database (supabase):** 사용자 정보, 채팅 내용 등 데이터를 영구적으로 저장합니다.
    *   **WebSocket Server:** 실시간 채팅 기능을 구현하기 위해 서버와 클라이언트 간의 양방향 통신을 지원합니다.

# 3. 핵심 기능 명세

미리보기에 포함된 모든 기능의 명세는 다음과 같습니다.

*   **로그인/회원가입:**
    *   사용자는 계정을 생성하고, 생성된 계정을 통해 서비스에 로그인할 수 있습니다.
    *   인증된 사용자는 서비스의 모든 기능에 접근할 수 있습니다.
*   **채팅방 개설:**
    *   사용자는 새로운 채팅방을 만들 수 있습니다.
    *   채팅방 생성 시, 채팅방의 이름이나 기타 설정을 지정할 수 있습니다.
*   **채팅방 참여:**
    *   사용자는 이미 개설된 채팅방 목록을 확인하고, 원하는 채팅방에 참여할 수 있습니다.
*   **메시지 전송 (텍스트/이모지):**
    *   사용자는 채팅방 내에서 텍스트와 이모지를 포함한 메시지를 전송할 수 있습니다.
    *   전송된 메시지는 해당 채팅방에 참여한 모든 사용자에게 실시간으로 보여집니다.
*   **좋아요:**
    *   사용자는 다른 사용자가 보낸 메시지에 '좋아요' 표시를 할 수 있습니다.
    *   '좋아요' 수는 해당 메시지와 함께 표시됩니다.
*   **답장:**
    *   사용자는 특정 메시지를 인용하여 답장할 수 있습니다.
    *   답장 메시지는 원본 메시지와의 관계를 시각적으로 표시합니다.
*   **내 메시지 삭제:**
    *   사용자는 자신이 작성한 메시지를 삭제할 수 있습니다.
    *   메시지가 삭제되면, 채팅방에서 해당 메시지 내용은 보이지 않게 처리됩니다.
